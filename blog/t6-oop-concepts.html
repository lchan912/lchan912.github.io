"Closures" Explained

Blocks, Procs, and Lambdas

June 7th, 2015

Have you ever felt that you deserved some sort of closure after an important relationship ended rather abruptly? I certainly have. Good thing, ruby took care of that by providing more closure tactics than I've actually ever encountered. In the world of computer science, "closure" is another way of saying a block of code that can be passed into methods like Array#each and Hash#map.

We'll be diving into three specific types of "closure" and explaining what they are and how they are used. First is block, which is probably the most familiar to you. Block are contained in either {} or do..end statements. But what if you want to reuse a block? How do you do it without retyping the entire block of code?

Introducing procs and lambdas, both of which allow you to set a block equal to a variable. That way you can reuse the block at any point in the code. Well, then what's the difference between procs and lambdas, you ask?

Both procs and blocks are limited in that you cannot use the explicit return statement within the closures. Else you will be returning from whatever method (e.g., Array#each) that called the blocks. Which means, you can only rely on the implicit return (i.e., last line of code) and return a single value. Lambdas, on the other-hand, allow the use of the explicit return and thereby gives you the option to return multiple values. 


      #Block example
      girlfriends = ['Amy', 'Kim', 'Joyce', 'Playstation']

      girlfriends.map do |g| 
          puts "#{g}, left me for no good reason." 
          #return "boo" #=> this would result in an "unexpected return" error
        end      
      
      #=> [nil, nil, nil, nil]   #The return value of 'puts' is nil
      

      #proc example
      girlfriends = ['Amy', 'Kim', 'Joyce', 'Playstation']
      reason = Proc.new do |g| 
          puts "#{g}, left me for no good reason." 
          #return "boo" #=> this would result in an "unexpected return" error
        end

      girlfriends.map(&reason)
      #=> [nil, nil, nil, nil]   #The return value of 'puts' is nil
      

      #lambda example
      girlfriends = ['Amy', 'Kim', 'Joyce', 'Playstation']
      reason = lambda do |g| 
          puts "#{g}, left me for no good reason." 
          return "boo"
        end

      p girlfriends.map(&reason)
      #=> ["boo", "boo", "boo", "boo"]
