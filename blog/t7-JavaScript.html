Fundamental Difference in Ruby hashs and JS objects

June 13th, 2015

My first realization of the difference in Ruby hash and JS object was when I was tasked to create a grocery list. As you can imagine, a grocery list usually contains items and quantities of items needed. In Ruby this can be done easily through hashes, as soon in the code below: 


      grocery_list = {milk: 1, apple: 6, butter: 2, flour: 1}
      #=> {:milk=>1, :apple=>6, :butter=>2, :flour=>1}

However, in JS instead of creating a hash, I created an object groceryList with names and value properties associated with items and quantities of items. Using object literal notation, the syntax for creating the JS object is similar to that of Ruby's. 


      var groceryList = {
        milk: 1,
        apple: 6,
        butter: 2,
        flour: 1
      }

Once I've created the grocery list (either as a hash in Ruby or as an object in JS), I can retrieve the key-value pairs or name-value properties using the following code. 


      #Ruby
      grocery_list[:apple]
      #=> 6
      

      //JavaScript
      return groceryList.apple;
      //=> 6

Additionally, if I need to remove or add an item to the grocery list, I execute the following code: 


    #Ruby
    #Remove item
    grocery_list.delete(:milk)

    #Add item
    grocery_list[:cereal] = 2
    
    #View hash
    grocery_list
    #=> {:apple=>6, :butter=>2, :flour=>1, :cereal=>2}

    

    //JavaScript
    //Remove item
    delete groceryList.milk;

    //Add item
    groceryList[cereal] = 2

    //View object
    return groceryList;
    // { apple: 6,
        butter: 2,
        flour: 1,
        cereal: 2}
