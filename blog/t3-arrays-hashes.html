Arrays vs Hashes

When to use what?

May 16, 2015

Arrays and hashes are two data structures that allow you to collect, view, and manipulate pieces of data in meaningful ways. For example, say you want to organize your contacts by their names and phone numbers. Both arrays and hashes can help you achieve that, except one would be more suitable than the other in this case.

Let's store the made-up contact list below so that we can easily retrieve the phone number of any given contact.

Jane Smith 234-533-5344
Lorri Ale 340-343-4449
Amber Joe 320-348-3822
Rocky Lee 284-239-3888
Joseph Young 394-939-9999
Alpha Dare 390-982-3332
Corey Tan 299-333-5555
Angie Jones 302-494-8842

If we were to use arrays to store the data and later retrieve a specific phone number, we execute the following ruby code: 


        contact_list = []
          contact_list.push(["Jane Smith", "234-533-5344"])
          contact_list.push(["Lorri Ale", "340-343-4449"])
          contact_list.push(["Amber Joe", "320-348-3822"])
          contact_list.push(["Rocky Lee", "284-239-3888"])
          contact_list.push(["Joseph Young", "394-939-9999"])
          contact_list.push(["Alpha Dare", "390-982-3332"])
          contact_list.push(["Corey Tan", "299-333-5555"])
          contact_list.push(["Angie Jones", "302-494-8842"])

We first created an empty array called "contact_list" and then populated it with contacts and their phone numbers wrapped in smaller arrays. Once the code runs, we can retrieve, say, Amber Joe's number with the following ruby code: 


        contact_list.each {|name, phone|
          puts phone if name == "Amber Joe"} # => 320-348-3822

The code has to loop through "contact_list" one name at a time until it finds Amber Joe and prints her phone number. Imagine if you were a social butterfly and had a contact list of 10,000 names. ruby would potentially have to loop through 10,000 contacts before finding the contact you want. Another more efficient way to manage your contacts is through hashes.

First populate your hashtable with the following ruby code: 


        contact_list = {}
          contact_list["Jane Smith"] = "234-533-5344"
          contact_list["Lorri Ale"] = "340-343-4449"
          contact_list["Amber Joe"] = "320-348-3822"
          contact_list["Rocky Lee"] = "284-239-3888"
          contact_list["Joseph Young"] = "394-939-9999"
          contact_list["Alpha Dare"] = "390-982-3332"
          contact_list["Corey Tan"] = "299-333-5555"
          contact_list["Angie Jones"] = "302-494-8842"

We created a hash called "contact_list" and populated the hash with key-value pairs. In this case, the keys are the contacts' names, which are each paired with the value of their phone number. In order to, once again, retrieve Amber Joe's number, we execute the following ruby code: 


        contact_list["Amber Joe"] # => 320-348-3822

Notice that in this case, we are not cycling through the hash to identify a match to the name "Amber Joe". Instead, the hash has indexed each phone number by a contact's name. Thereby allowing you to simply provide a name (i.e., the key or index value) and the hash point to the corresponding phone number (i.e., the value). This concept is called key-value pairs.

Overall hashes are more efficient and effective for use cases where objects need to be associated with proprieties or values, like the English vocabulary and their definitions. Arrays are helpful when being able to iterate through data is important and when storing different types of data in a data set.
